Starting with V0.9.0, you can have HTML documentation created for the whole XWorkplace code. This done using my own xdoc utility, which can be found in the TOOLS\ directory after XWorkplace has been built for the first time.

xdoc is capable of parsing C/CPP source code and header files to some extent. When certain codes are found in a comment in the sources (e.g. @@), that comment is assumed to document some function and will be considered for HTML output.

After XWorkplace has been built for the first time, call createdoc.cmd in the main directory, which will create the HTML documentation in a new subdirectory of the code tree HTML\. Open index.html then to get an index of all XWorkplace source files.
