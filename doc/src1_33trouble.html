1. If you get errors from the makefiles, make sure you have set up the environment variables right.

  2. Also, you need to check out the WarpIN source tree for the helpers code.

  3. If you get something like "File XWPHOOK.DLL is in use", the XWorkplace hook DLL is currently loaded and running. While all other executables are automatically unlocked and replaced by the makefiles even while they're running, this cannot be done with the hook DLL, because then you'd only be able to reload that DLL by rebooting (it will _never_ get released otherwise).

    Use WatchCat or some other process killer to kill XWPDAEMN.EXE, which will automatically unload the hook DLL. Then, after the build, restart the daemon by executing XWPDAEMN -D (to circumvent the error message box which comes up otherwise).

    Warning: If you don't restart the daemon before restarting the WPS, the XWorkplace startup folder will get processed again after the WPS restart.

  4. If you cannot install XWorkplace after the build, you have probably built the debug version, but the PMPRINTF DLLs are not on your LIBPATH. See "Debugging" for details.
