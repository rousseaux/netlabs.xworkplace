1. Compiler. With V0.80, I have switched to IBM VisualAge C++ 3.0 to develop XWorkplace. For VAC++ 3.0, fixpak 8 is strongly recommended, because the original crashes frequently with the complex SOM header files.

    As of V1.0.9, XWorkplace should also compile successfully using VAC 3.6.5. Other compilers are presently not supported.

  2. You need some OS/2 Developer's Toolkit for all the SOM header files and the SOM compiler.

    It is recommended to use the Warp 4.5 toolkit. Since compatibility with Warp 3 was given up with XWorkplace 1.0.1, there is no requirement to use the Warp 3 toolkit anymore. Unfortunately, most versions of the Warp 4.5 toolkit have a broken SOM compiler, nmake.exe, and rc.exe.

    To solve this problem, you can either use the toolkit that comes with eComStation 1.2R and later, or use the following work-around:

    • Either... copy the emitter DLLs from the Warp 4 toolkit, except for EMITIR.DLL. See the EMIT*.DLL files in the SOM\LIB\ subdirectory of the toolkit.

    • Or... change the SMBASE variable in config.in to point to a directory containing an earlier version of the SOM compiler.

    • Copy nmake.exe from the Warp 3 toolkit, or use the one from VAC++.

    • Use the version of rc.exe that comes with OS/2 and eComStation.
  3. IBM NMAKE is our make utility. This comes with VAC, so no problem there.

  4. You will also need to check out the "xwphelpers" CVS archive from Netlabs. It contains code which is shared by both XWorkplace and WarpIN. Note: you don't need the WarpIN sources to compile XWorkplace, only "xwphelpers".

  5. Only if you enable debugging code through XWP_DEBUG in config.in, you will need the full PMPRINTF package by Dennis Bareis.

    Check the "Debugging" section for details.
