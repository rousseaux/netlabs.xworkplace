&xwp; requires at least OS/2 Warp 3. It will NOT run on previous versions, for it calls OS/2 API functions not present on OS/2 version 2.x and before.

Over the years, I have tested XFolder/&xwp; with the following configurations:

  • OS/2 Warp 3, no fixpaks installed
  • OS/2 Warp 3, fixpak 35 installed
  • OS/2 Warp 4, no fixpaks installed
  • OS/2 Warp 4, fixpaks 5, 7, 10, 11, 13, 14, and 15 installed
  • OS/2 Warp Server for e-business ("Aurora"), no fixpaks installed
  • OS/2 Warp Server for e-business ("Aurora"), fixpak 1 and 2 installed
I was reported that installation fails under Warp 3, fixpak 31.

&xwp; V0.9.16 has been tested with eComStation 1.0 only, but should work on Warp 4 as well. I am no longer testing &xwp; on Warp 3, even though it should work, since no Warp-4 specific APIs are used.

In general, I strongly recommend installing newer fixpaks. If you're using Warp 4, I can recommend FP15 with one of the newer testcase kernels, which solves most of the problems from FP13/14. Or get eComStation. See the FAQ section for details.
