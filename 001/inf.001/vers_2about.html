With the release of &xwp; V0.9.0, the version numbering format has been altered. Similar to the Linux version numbering, we now have three sub-digits in the version number, which is now in the format major.minor.revision.

As with Linux, odd minor numbers are used for "developer releases" which are not guaranteed to be stable. After several developer releases, if the code is considered to be stable, the next version will get an even minor number.

So version 1.0.0 means that the code is considered ready for general use.
