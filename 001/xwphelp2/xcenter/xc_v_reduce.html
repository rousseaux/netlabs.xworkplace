If "Reduce Desktop workarea" is enabled, the PM desktop's workarea will be reduced by the size of this &xcenter;.

Enabling this setting for any &xcenter; will have the effect that maximized windows will not be maximized to the full size of the screen anymore, but only to the screen size minus all open &xcenter;s that have this setting enabled.

This is equivalent to &warpcenter;'s "Put on top of maximized windows" setting, whose name isn't really that correct.

This cannot be used together with "auto-hide".

Notes:

  • This probably doesn't work on Warp 3.

  • This setting is global and will affect the entire system. It can lead to the same problems as with &warpcenter;'s setting (most notably scroll bars on the WPS Desktop), so use this with care.

  • Do not use this setting on the &xcenter; if you have the corresponding &warpcenter; setting enabled also. The &xcenter; cannot check for whether the &warpcenter; has this enabled as well, and this will probably lead to problems.
