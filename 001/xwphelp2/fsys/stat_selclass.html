In this window you may select the object class for which you wish to set status bar information when only one object is selected in a folder.

This window shows you all the installed WPS classes to give you an overview about which class is derived from which.

For determining status bar codes however you may only select a class which can process this information. Such classes are printed in black, while classes which don't support this appear in gray and cannot be selected.

The tree view has been extended so that all classes which support status bar information are initially visible.

The generic status bar information is determined by the class XFldObject. That is, if you select a single object for which no class defines more specific status bar information (an example for this would be a printer object), the codes of the XFldObject class are used.

More specific keys are however provided by the classes XFldDisk (for "disk" objects, as in the "Drives" folder), WPProgram (for abstract program objects), WPFileSystem (for folders and data files), and WPUrl.

Note that status bar information is independent of the additional &xwp; classes. That is, it is not only &xwp; classes which support status bar information, but on the other hand, not all &xwp; classes support this either.
