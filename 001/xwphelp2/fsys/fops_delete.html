Use the "Delete" menu item to remove the selected objects from a folder.

Per default, selecting "Delete" will not instantly destroy the object yet, but move it into the &xwp; trash can instead. This allows you to recover the object if you accidentally selected "Delete". To permanently destroy all objects that have been moved into the trash can, select the "Empty trash can" menu item from the trash can's context menu.

As a shortcut, you can also press the "Delete" key on your keyboard to have the selected objects deleted.

If you want to permanently destroy the object immediately, without moving it to the trash can first, hold down the "Shift" key when selecting the "Delete" menu item or pressing the "Delete" key. Be warned that this operation normally cannot be undone: the selected objects are destroyed for good.

Depending on the settings on the "Confirmations" page in the "Workplace Shell" object, &xwp; will display a confirmation message before permanently deleting objects.

Note that there are ways to recover file-system objects that have been accidentally deleted with the use of the UNDELETE command on the &os2; command line. However, this requires that support for this is enabled first. See the documentation of the UNDELETE command in the &os2; Command Reference.

You can disable &xwp;'s delete replacements on the "Features" page in the "&xwp; Setup" object. The above description applies to the case where you have both "Replace true delete" and the &xwp; trash can enabled. If you only enable the trash can, the WPS will still perform the permanent destruction when the "Shift" key is pressed. If you only enable "Replace true delete", all delete operations will be performed without moving the objects into the trash can first.
