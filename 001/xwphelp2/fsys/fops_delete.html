Use the "Delete" menu item to remove the selected objects from a folder.

Per default, selecting "Delete" will not instantly destroy the object yet, but move it into the &xwp; trash can instead. This allows you to recover the object if you accidentally selected "Delete". To permanently destroy all objects that have been moved into the trash can, select the "Empty trash can" menu item from the trash can's &popmenu;.

As a shortcut, you can also press the "Delete" key on your keyboard to have the selected objects deleted.

If you want to immediately delete the selected objects, without moving them to the trash can first, hold down the "Shift" key when selecting the "Delete" menu item or pressing the "Delete" key. Be warned that this operation normally cannot be undone: the selected objects are destroyed for good.

Notes:

  • Depending on the settings on the "Confirmations" page in the "Workplace Shell" object, &xwp; will display a confirmation message before permanently deleting objects.

  • If you prefer to always immediately delete objects, instead of moving them to the trash can first, enable the respective setting on the first properties page in the trash can's properties notebook.

  • There are ways to recover file-system objects that have been accidentally deleted with the use of the UNDELETE command on the &os2; command line. However, this requires that support for this be enabled first. See the documentation of the UNDELETE command in the &os2; Command Reference.

You can disable &xwp;'s delete replacements on the "Features" page in the "&xwp; Setup" object.
