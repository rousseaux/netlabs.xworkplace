Under &os2;, programs run in sessions. A session consists of one or more processes and can run in a window or in full-screen mode. In addition, every session has a special type, depending on the operating system that a program was written for.

Even though programs are normally written to run under a specific operating system, &os2; has built-in support to be able to execute programs for other operating systems too. &os2; supports the following session types:

  • The Presentation Manager (PM) session. PM is the graphics subsystem of &os2; and is responsible for managing multiple windows on the screen, supporting graphics output such as icons and pictures, mouse and keyboard input, and standard controls such as menus and title bars. For example, the help window which displays the text you are currently reading is a PM window. The &os2; Workplace Shell, which displays your Desktop, is a PM program too.

    PM programs always run in the one and only PM session that is started when the system starts up. You cannot start a second PM session. However, you can extend your screen space with the use of the &pgr;, which allows you to switch between several virtual desktops.

  • Many OS/2 programs however do not use PM, but run in text mode instead. Any OS/2 text-mode program, including command files (.CMD extension), will be started in a separate session that can either be a windowed or full-screen.

    Windowed command prompts are visible on your Desktop as well as if they were part of the PM session, so you can still see all other windows that are open.
    By contrast, an full-screen session will occupy the entire screen. Your other windows are still open and the other programs keep running, but in order to see them, you have to switch back to the PM session using the Alt+Esc or Ctrl+Esc system hotkeys.

  • &os2; can also run the large majority of DOS programs. DOS is the original operating system for personal computers that was shipped with the first IBM PC. DOS programs, including batch files (.BAT extension), usually run in text-mode and can run windowed or full-screen as well.
    Windowed DOS sessions are visible on your Desktop as well.
    Full-screen DOS sessions will occupy the entire screen, similarly to other full-screen sessions.

  • &os2; can also natively execute the great majority of 16-bit Windows applications written for Microsoft Windows 3.x.
    These run in so-called &winos2; sessions which can be windowed or full-screen as well. If a &winos2; program is started as a window, it runs seamlessly on the &os2; Desktop as if it was a PM program, even it will have a slightly different appearance. However, you can also select to run a &winos2; program in a separate full-screen session.
