This page allows you to fine-tune the behavior of a single &xwp; startup folder.

Press "Undo" to restore the settings that were in use before you opened this notebook page.

Press "Default" to restore the settings to the installation state.

The following settings are available:

  • The "Initial delay" is the time that the startup folder will wait (after the Desktop has been populated) before it starts processing.

    Note: This setting affects all &xwp; startup folders.

  • The "Per-object delay" specifies how long the startup folder waits after each object in the folder has been opened. This is different from the default &os2; startup folder, which starts all contained objects almost at once, frequently causing WPS hangs at startup.

    Note: This setting affects all &xwp; startup folders.

  • If "Show status window" has been checked, you'll get a status window with a progress bar while the &xwp; startup folder is being processed. That window also has a "Cancel" button to stop startup processing.

  • If "Reboot only" is chosen, the folder's contents are only started when the WPS is started the first time after system bootup. (If you use &xwp;'s "Restart Desktop" feature, you can override this setting for a Desktop restart.) This is useful if you have objects in your startup folder which keep running in the background even if the WPS is restarted.

  • By contrast, if "Every Desktop restart" is selected, the folder is processed at every Desktop restart -- that is, even if you have used "Restart Desktop", or if the WPS restarted because of a crash. This might be useful for starting Desktop objects which are not in a separate process, such as the &xcenter;.
