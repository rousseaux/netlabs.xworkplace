The "Objects" page allows to recreate the most important default objects, both of the WPS and &xwp;, in case any of these got lost somehow or were accidentally deleted.

This page has two "menu buttons", one for the default Desktop objects, and one for the default &xwp; objects. When you press one of these buttons, a menu pops up which contains a list of the objects that can be recreated.

These menus only contain the objects which are difficult to recreate, mostly because they are instances of special WPS classes which were designed specifically for one system object. The menu does not contain objects which you can safely recreate yourself using regular folder templates.

If &xwp; finds out that an object exists already, the respective menu item is disabled, and the path of the folder where the existing object resides is displayed in the menu.

Otherwise, the menu item is enabled, and &xwp; will create the respective object when you select the menu item. The object will be created on your Desktop, but you can then move it anywhere you like.

Note that &xwp; uses object IDs to determine whether an object exists already. If the object exists, but does not have the correct ID set, it is considered to be non-existant, and the menu item is enabled anways. Be careful when recreating those objects, because some objects were designed to be unique on the system.
