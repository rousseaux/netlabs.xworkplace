An &xwp; "font object" represents a font file which is currently installed. Such a font object can only appear in the open Fonts folder.

Note: In this context, the term "font" refers to a font face, which is physically represented by a font file. For most fonts, each font family is represented by several faces. For example, for the "Times New Roman" family, you will have four font files (one for "Regular", "Bold", "Italics", and "Bold Italics", respectively). Only if no face is found for a certain format (for example, if you want bold text in a word processor, but no bold face exists for the current font), &os2; will imitate bold and italic formats in its own graphics engine, which in most cases will look quite ugly.

Even though most applications (and the standard &os2; font dialog as well) will only enumerate the various font family name and automatically find the correct font file for a format that has been selected, the Fonts folder will list all installed faces separately. You can see the respective families in the Fonts folder's details view however.

You can install fonts into the system by dragging font files and dropping them into an open Fonts folder. Reversely, you can uninstall a font by selecting "Uninstall" from a font object's &popmenu;, after you have opened the Fonts folder.

Font objects have a special "Sample" view, which is opened when you double click on a font object. See Font sample view for details.
