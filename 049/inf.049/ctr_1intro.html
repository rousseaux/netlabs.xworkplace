Beginnend mit V0.9.7 enthält &xwp; einen Ersatz für &warp4;s &warpcenter;. Dieser trägt den Namen "&xcenter;".

Technisch gesehen handelt es sich nicht um einen &warpcenter;-Eratz... es ersetzt die WPS-Klasse "SmartCenter" in erster Linie deswegen nicht, weil sie gänzlich undokumentiert ist. Stattdessen stellt das &xcenter; eine vollkommene Neuimplementierung dar. Sie ist nicht mit dem &warpcenter; kompatibel und ünimmt dessen Konfiguration nicht, aber sie bietet annähernd diegleiche Funktionalität... und mehr.

Im Vergleich zum &warpcenter; bietet Ihnen das &xcenter; die folgenden Vorteile:

  • Es ist voll konfigurierbar. Sie können dem &xcenter; beliebige Einträge hinzufügen, während das &warpcenter; immer dieselben festen Einträge an denselben festen Positionen enthält.

    Die Einträge des &xcenter; werden "Widgets" genannt. Sie können dem &xcenter; Widgets in beliebiger Reihenfolge hinzufügen.

    Details finden Sie unter "Widgets hinzufügen und entfernen".

  • Auf Ihrem System darf sich mehr als ein &xcenter; befinden. Jedes &xcenter; speichert seine Widgets und deren Einstellungen getrennt ab, so daß diese nie in Konflikt geraten können.

  • Das &xcenter; bietet Ihnen mehr Anzeigestile als das &warpcenter;. Sie können Schriftarten und Farben auf die Widgets ziehen, welche diese Einstellungen dann individuell behalten. Zudem besitzt das &xcenter; eine Funktion "automatisch Verstecken", mit der es sich auf eine kleine Leiste am unteren Bildschirmrand verkleinert, ähnlich der Taskbar bei Windows 95. Diese Anzeigeoptionen können sie im Einstellungsnotizbuch des &xcenter;s festlegen.

  • Das &xcenter; unterstützt Plugin-DLLs. Jeder Programmierer kann dem &xcenter; neue Widgettypen hinzufügen, indem er/sie eine DLL schreibt und sie im Unterverzeichnis plugins\xcenter des &xwp;-Installationsverzeichnisses plaziert.

    Details finden Sie unter "Über Plug-in-DLLs".

  • Das &xcenter; läft innerhalb der WPS in einem separaten Thread und kann so mit einer höheren Priorität als andere Programme laufen. Die Priorität kann im Einstellungsnotizbuch des &xcenter;s festgelegt werden.

    Wenn damit auch die Probleme mit der Single Input Queue (SIQ) des PM nicht behoben werden können, so behalten Sie dennoch die Möglichkeit, auf das &xcenter; zuzugreifen, wenn ein Programm die gesamte CPU-Zeit in Beschlag nimmt.
