Die xfix Menüleiste stellt Kommandos bereit die man auf die meisten oder alle aufgelisteten Handles anwenden kann. Siehe Popup Menü und Tastaturkürzel für weitere Details in Bezug auf die Kommandos die auf einzelne Handles angewendet werden können.
  • xfix Menü

    • Puffer nochmals verarbeiten durchsucht noch einmal den Handles Puffer so wie er gegenwärtig ist und erzeugt die Handles Liste neu. Die Handles werden also nicht von OS2SYS.INI neu geladen. Dies ist hilfreich wenn sie einige Handles gelöscht haben und die Auswirkungen der Löschung sehen wollen. Wenn plötzlich Handles auftauchen die einen Fehler anzeigen, vorher aber in Ordnung waren, so speichern sie bitte ihre Änderungen nicht dauerhaft ab!
    • Zurück in OS2SYS.INI schreiben... schreibt die geänderte Handletabelle zurück in die OS2SYS.INI Datei. Die Änderungen an der Handleliste werden erst dauerhaft gespeichert wenn sie diesen Menüeintrag wählen. Wenn Handles zum Löschen ausgewählt sind werden sie um Bestätigung des Vorgangs gebeten bevor die Änderungen dauerhaft abgespeichert werden.

      Seien sie sich bewußt daß dies keinen Effekt hat wenn die WPS im Hintergrund läuft weil die WPS eine interne Kopie der Handletabelle verwaltet und es wahrscheinlich ist, daß ihre Änderungen wiederrum von der WPS überschrieben werden. Dies ist der Grund warum die WPS nicht aktiv sein sollte wenn sie die Änderungen dauerhaft abspeichern wollen. Siehe auch Die Grundlagen.

    • xfix beenden beendet das Programm.
  • Anzeige Menü

    • Objekt-IDs... zeigt ein neues Fenster an welches alle WPS Objekte auflistet denen eine Objekt-ID zugeordnet ist. Es werden für jedes Objekt die ID, das Handle sowie der Langname (voller Dateipfad) angezeigt. Die Liste kann nach den entsprechenden Feldern sortiert werden. Dieses Fenster dient nur ihrer Information, es können keine Aktionen ausgeführt werden.
  • Sortieren nach Menü

    Dies erlaubt es die Handles nach den verschiedenen Spalten zu sortieren. (Siehe Die Handletabelle für eine Beschreibung der Spalten.) Wenn sie nach Index sortieren wird ihnen genau die Reihenfolge angezeigt wie die Handle in der Handletabelle in OS2SYS.INI zu finden sind (mit entsprechend ansteigenden Knoten Offsets). "Sortieren nach Index" ist die Standard Sortierreihenfolge nach dem Start von xfix.

  • Aktionen Menü

    • Suche ungültige Dateien prüft alle Handles in der Handletabelle ob sie auf existierende Dateien verweisen. Jeder ungültige Eintrag wird als solcher markiert und der Hinweis "Datei nicht gefunden" als Status angezeigt. Dies passiert automatisch da dieser Prozess eine Weile dauern kann. Siehe Schritt für Schritt für eine typische Benutzung dieses Menüeintrags.
    • Auto-Schutz schützt das Handle vor unbeabsichtigtem Löschen all der Dateien bzw. Ordner die für das korrekte Funktionieren der WPS unabdingbar sind und die sie höchstwahrscheinlich auch nicht löschen wollen. Es wendet den Schutz auf folgende Dateien an:
      • Dateien mit diesen Dateiendungen: .EXE .COM .CMD .BAT
      • Ordner die abstrakte Objekte enthalten wie zum Beispiel Programmobjekte oder Schattenobjekte
      • Dateien oder Ordner denen eine Objekt-ID zugewiesen wurde
      • Einträge für Laufwerke und Wurzelordner.
      Jeder geschützte Eintrag wird markiert als . Jeder übergeordnete Ordner (der im Dateipfad liegt) wird markiert als und das zugehörige Handle ist ebenso vor dem Löschen geschützt.
    • Schutz aufheben (alle) entfernt den Löschschutz von allen Einträgen.
    • Jetzt löschen... zeigt einen Bestätigungsdialog an um nachzufragen ob Einträge die als markiert sind gelöscht werden sollen. Wenn sie mit Ja antworten werden diese vom Fenster und aus dem Speicher entfernt. Sie werden nicht aus OS2SYS.INI gelöscht bis sie Zurück in OS2SYS.INI schreiben... aus dem xfix Submenü auswählen.

      Wenn das Löschen von Handles dazu führen würde das andere Handles "verwaisen" (und damit auch dauerhaft ungültig würden) werden sie eine Warnung erhalten. Sie sollten mit Nein antworten um die Handles vom Löschen auszunehmen die das Problem hervorgerufen haben.

  • Auswählen Menü

    • Alle ungültigen auswählen selektiert alle Handles die, aus welchem Grund auch immer, als ungültig markiert sind. Sie können mit der rechten Maustaste auf den selektierten Eintrag klicken und "Löschen" vom &popmenu; wählen.
    • Nach Name auswählen... zeigt einen neuen Dialog an der es ihnen erlaubt mehrere Handles auf einmal an- bzw. abzuwählen. Sie können alle Handles an- bzw. abwählen oder sie können eine Suchmaske ähnlich der die auf einer OS/2 Kommandozeile benutzt werden kann benutzen, die Platzhalter ? and * können also benutzt werden um einen oder mehrere Zeichen zu substituieren. Die Suchmaske "*.txt" würde also alle Handles mit der Namensendung ".txt" an- bzw. abwählen. Der Dialog bleibt geöffnet bis sie den "Schließen" Knopf auswählen oder die Esc Taste betätigen. Auf diese Weise können Auswahlen nach unterschiedlichen Suchmasken eine nach der anderen angewendet werden.

      Die Dateimaske wird nur auf Kurzer Name angewendet. Es ist also nicht möglich die Handles aller Objekte die in einem bestimmten Ordner liegen auf diese Weise auszuwählen.
