Dalla finestra a cascata "Salva file INI" Š possibile scegliere il metodo utilizzato da &xwp; per salvare i profili utente e di sistema (nei file OS2.INI e OS2SYS.INI come valore assunto) durante la chiusura. Questa funzione non Š utilizzata durante il riavvio della Scrivania.

Questa impostazione Š stata aggiunta a partire dalla V0.9.5 per risolvere i problemi, segnalati da numerosi utenti, che si verificavano quando la &xshutdown; si trovava a gestire file INI di grandi dimensioni (parecchi Mb). Le funzioni relative ai profili (INI) di &os2;, sfortunatamente, hanno serie difficolt… a lavorare con file di dimensioni davvero grandi; sembra infatti che queste API esauriscano rapidamente la memoria condivisa.

Per questo motivo &xwp; dispone di un sistema alternativo per salvare i file INI sul disco. Qui si pu• scegliere che sistema utilizzare:

  • "Nuovo metodo": Utilizza per il salvataggio dei file INI le nuove funzioni profilo introdotte con &xwp; V0.9.5 che non utilizzano memoria condivisa; Š perci• meno probabile che possano verificarsi errori durante il salvataggio dei file INI. E' l'impostazione consigliata.

  • "Metodo tradizionale": Utilizza le funzioni profilo di &os2;. E' il sistema usato da XFolder e dalle versioni di &xwp; precedenti la V0.9.5 e notoriamente pu• provocare problemi in presenza di file INI di grandi dimensioni. Si veda la sezione "FAQ" della Guida ad &xwp; per dettagli.

  • "Non salvare i file INI": Alla chiusura del sistema i file INI non vengono salvati: si perderanno perci• le modifiche pi— recenti. E' l'impostazione da usare solo se nessuno degli altri due metodi funziona come si deve.

    &os2; salva automaticamente i file INI sul disco ogni pochi minuti, perci• si perderanno tutte le impostazioni successive all'ultimo salvataggio automatico. E' sostanzialmente equivalente all'utilizzo dei tasti Ctrl-Alt-Canc, ed influenza tutte le informazioni conservate da &os2; nei file INI (oggetti della Scrivania, tipi di catattere, colori, impostazioni relative alle applicazioni). E' insomma un'impostazione da usare come estrema risorsa.

    La posizione delle icone e le altre informazioni conservate negli attributi estesi degli oggetti verranno comunque salvate correttamente.
