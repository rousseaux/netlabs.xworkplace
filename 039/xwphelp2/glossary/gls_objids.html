La Workplace Shell prevede che ciascun oggetto possa avere una propria IDentit… oggetto opzionale, che rende unico ed identifica l'oggetto all'interno del sistema. Ogni oggetto pu• avere una ID di questo tipo, indipendentemente dalla classe.

Per definizione, le ID degli oggetti della Scrivania sono una stringa caratteri racchiusa tra parentesi ad angolo. Per esempio, la &link_syssetupfdr; ha <WP_CONFIG> come ID oggetto. Le applicazioni, utilizzando tale ID, possono fare riferimento all'oggetto senza bisogno di scoprire in quale cartella esso risieda effettivamente.

Molte delle ID oggetto vengono assegnate dal programma di installazione di &os2; durante l'installazione del sistema e sono destinate a non essere pi— modificate (queste ID non devono essere cambiate innanzitutto perch‚ diverse applicazioni, nonch‚ la WPS stessa, fanno affidamento su di esse per rintracciare gli oggetti).

Tuttavia, nel sistema vi sono molte ID oggetto oltre quelle assegnate durante l'installazione di &os2;. Numerosi programmi di installazione, infatti, assegnano ID ai propri oggetti per poterli trovare e rimuovere in un secondo tempo.

&xwp; permette di vedere, e modificare, la ID di un oggetto dalla pagina "Icona" presente in tutti i blocchi appunti impostazioni. Si presti la massima cautela nell'apportare modifiche alle ID oggetto!
