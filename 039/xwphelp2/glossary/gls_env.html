Praticamente tutti i sistemi operativi prevedono che i programmi lavorino all'interno di un ambiente,, definito attraverso una serie di variabili ambiente, che si presentano nella forma CHIAVE=VALORE.

Definire l'ambiente ha diverse funzioni:

  • In molti casi definisce come il sistema operativo deve agire rispetto al programma che sta facendo funzionare. Per esempio, i percorsi di sistema sono definiti attraverso variabili ambiente. Se si modificano queste variabili prima di avviare il programma, si modifica il percorso che il sistema operativo segue per trovare i file richiesti dal programma.

    Le variabili ambiente sono riportate in &os2; - Riferimento per i comandi (CMDREF.INF) e sono generalmente definite all'avvio del sistema attraverso il file &cfgsys;.

Per sapere se vi sono variabili definite per un dato programma, e nel caso quali sono, bisogna far riferimento alla documentazione dell'applicazione. &os2; non sa nulla di queste variabili: si limita a fornire la struttura necessaria per impostarle.
Per leggere o impostare una variabile ambiente dalla riga di comando usare l'istruzione SET.

E' possibile specificare una variabile ambiente delimitandone a destra e a sinistra il nome con un segno di percentuale (%). Tale espressione verr… rimpiazzata dinamicamente con il valore della variabile.

Per esempio, per aggiungere la directoru C:\NUOVADIR all'elenco corrente delle directory nel percorso PATH si digiti il comando 

SET PATH=%PATH%;C:\NUOVADIR

Usando il comando SETLOCAL si crea una copia locale dello spazio ambiente, della directory e dell'unit… corrente, che permette di apportare modifiche temporanee. Le impostazioni si possono poi ripristinare con il comando ENDLOCAL.
