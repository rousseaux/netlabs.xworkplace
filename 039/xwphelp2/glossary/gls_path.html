Ogni oggetto file-system (file o cartella) ha un percorso associato, che indica al sistema dove l'oggetto Š conservato, in modo che le applicazioni vi possano far riferimento.

Il percorso Š formato dall'insieme delle directory che si devono aprire per raggiungere quel dato file (le directory rappresentano fisicamente le &link_folders; sul disco).

Con percorso qualificato di un oggetto si intende il nome dell'unit… seguito da due punti (:), un backslash iniziale (\) per indicare l'indirizzario radice e poi l'elenco delle directory da aprire per raggiungere l'oggetto, ciascuna separata da un backslash.

Se il percorso punta ad un file, ad es. un file dati o un file programma, in fondo vi sar… il nome del file. Per esempio, &eexe; &os2; si trova nella directory OS2 sull'unit… di avvio. Il suo percorso completo Š 

    X:\OS2\E.EXE
dove X: Š la lettera dell'unit… di avvio.

&os2; prevede diversi percorsi di sistema che specificano dove cercare un file se non ne viene specificato il percorso qualificato. Si veda "Percorsi di sistema" per spiegazioni ulteriori.
