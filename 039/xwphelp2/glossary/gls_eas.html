Gli attributi estesi (EA) sono una funzione speciale di &os2;, che permette di immagazzinare informazioni aggiuntive negli oggetti file-system (siano essi file o cartelle). Queste informazioni aggiuntive non vengono salvate nei dati dell'oggetto ma vengono conservate assieme ad esso nel file system.

Differentemente dai vecchi bit attributi file, gli attributi estesi sono una lista di parole chiave con un valore collegato, in formato testo o binario.

Le applicazioni possono definire e poi utilizzare proprie parole chiave. Talune parole chiave sono definite dal sistema ed usate, per es., dalla Workplace Shell. Sono i cosiddetti "attributi estesi standard", caratterizzati dal fatto che il nome della parola chiave che li indica inizia con un punto. Si veda "Attributi estesi standard" per ulteriori informazioni.

Per visualizzare gli attributi estesi correntemente definiti per un certo oggetto nel file-system si apra il suo blocco appunti impostazioni e si selezioni la seconda pagina "File".

Note:

  1. La dimensione massima per gli EA di un oggetto file-system Š limitata a 64 KB.

  2. Tutti i file system in dotazione ad &os2; (HPFS, JFS e FAT) supportano gli attributi estesi, il che potrebbe non essere vero nel caso di file system forniti da terzi o di unit… di rete su server remoti. Gli EA vanno persi se si copiano file su file system che non li supportano.

  3. Se si condivide un volume tra &os2; ed un sistema operativo che non riconosce gli attributi estesi, pu• darsi che l'altro sistema operativo, nell'effettuare operazioni sui file, rovini gli EA salvati da &os2;. E' un'eventualit… estremamente probabile con il file system FAT, che Š letto praticamente da ogni sistema operativo.

  4. Per sapere quanto spazio Š complessivamente utilizzato per gli EA su un dato volume, selezionare "Controllare disco" dal suo menu contestuale.
