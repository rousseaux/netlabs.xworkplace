&os2; prevede diversi percorsi di sistema, che servono a specificare dove il sistema va a cercare i file se non viene definito esplicitamente il percorso per raggiungerli.

Se, per esempio, un certo file eseguibile si trova in una delle directory specificate nel percorso di sistema PATH, tale eseguibile verr… trovato anche se non si definisce il percorso da seguire per raggiungerlo. E' per questo motivo che per avviare l'&eexe; &os2; Š sufficiente digitare e da una richiesta comandi &os2;: E.EXE si trova nella directory OS2 dell'unit… di avvio, che Š compresa nel PATH.

Eccezion fatta per LIBPATH, si tratta di variabili ambiente che sono definite all'avvio del sistema attraverso il file &cfgsys; e che possono poi essere modificate per ogni sessione.

Si pu• modificare il percorso di sistema corrente dalla pagina "Percorsi di sistema" nell'oggetto "Kernel &os2;".

Sono disponibili i seguenti percorsi di sistema:

  • LIBPATH: il percorso delle librerie, che viene utilizzato quando &os2; cerca di caricare una libreria a collegamento dinamico (DLL). E' opportuno che questo percorso contenga le directory di sistema di &os2;, insieme ad un punto ("."), che indica la directory corrente.

    Si veda la documentazione in &os2; - Riferimento per i comandi per i dettagli.

    Si noti che questo Š il solo percorso di sistema che non viene definito tramite una variabile d'ambiente, e pertanto non pu• essere cambiato dalla linea di comando. Si usino le variabili BEGINLIBPATH e ENDLIBPATH per cambiamenti temporanei (non supportati tramite questa pagina).

  • SET PATH: il percorso di ricerca per gli eseguibili (file .EXE, .COM, .BAT, .CMD).

    Si veda la documentazione in &warp; - Riferimento per i comandi per i dettagli.

  • SET DPATH: il percorso di ricerca per i file dati. L'uso di questo percorso dipende dalle applicazioni; alcune lo usano, altre no.

    Si veda la documentazione in &os2; - Riferimento per i comandi per i dettagli.

  • SET BOOKSHELF: questo percorso viene utilizzato da VIEW.EXE per trovare le guide in linea (.INF). Anche se un file .INF pu• essere visualizzato senza essere compreso in questo percorso, il comando "Ricercare..." di VIEW.EXE esaminer… solo gli argomenti dei file presenti in questo percorso.

  • SET HELP: questo percorso viene usato dalla funzione di aiuto di PM per mostrare i pannelli di aiuto (file .HLP). Alcune applicazioni non specificano il percorso completo dei file di aiuto, e in questo caso Š necessario che il file si trovi in questo percorso. &xwp; non ha necessit… che si specifichi qui la sua directory.
