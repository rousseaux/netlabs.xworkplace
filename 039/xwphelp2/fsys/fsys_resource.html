La pagina "Risorse" Š presente nell'oggetto impostazioni di tutti i file eseguibili e mostra informazioni aggiuntive relative al modulo dell'eseguibile.

Le risorse sono oggetti dati di sola lettura accessibili dinamicamente al momento dell'esecuzione.

Gli oggetti risorse sono inseriti nel file eseguibile dal compilatore di risorse (RC.EXE), fornito con &os2;. L'eseguibile pu• essere sia il modulo principale dell'applicazione (file .EXE) sia una libreria a collegamento dinamico(DLL).

Le risorse possono essere caricate in qualsiasi momento utilizzando la coppia "tipo" e "ID". In genere, a differenza del codice applicazione principale, le risorse vengono caricate solo quando effettivamente necessarie. Questo rende pi— rapido l'avvio iniziale dell'applicazione ed inoltre riduce il consumo di memoria.

Sono usate comunemente per separare il supporto lingue nazionali (NLS) dal codice effettivo dell'eseguibile. In questa maniera, le applicazioni possono essere tradotte senza che sia necessario ricompilare il codice sorgente (il supporto lingue in &xwp;, per esempio, funziona appunto in questa maniera).

Esistono i seguenti tipi di risorse:

  • Mouse pointer shape: file icona o puntatore (formato .ICO/.PTR).

  • Bitmap: bitmap &os2; (.BMP).

  • Menu template: definizione menu PM; pu• essere sia una barra menu che un &popmenu;e.

  • Dialog template: definizione per finestre di dialogo PM.

  • String table: tabella stringhe.

  • Font directory, Font: tipo di carattere bitmap PM.

  • Accelerator table: tabella acceleratori PM (tasti rapidi, scorciatoie da tastiera).

  • Binary data: dati binari in formato non definito.

  • Error message table: tabella di messaggi errori.

  • Dialog include-file name: nome file del file "include" della finestra di dialogo.

  • Virtual key table: chiave per tabelle VKEY. Non mi Š chiaro a che serva.

  • Key table: chiave per tabelle UGL. Non mi Š chiaro a cosa serva.

  • Character table: tabella glifo-carattere. E' usata probabilmente per i tipi di carattere.

  • Display information: informazioni visualizzazione schermo. Mai incontrato.

  • Short-form function key area: area chiavi funzione, forma breve. Non mi Š chiaro a che serva.

  • Long-form function key area: area chiavi funzione, forma lunga. Non mi Š chiaro a che serva.

  • Help table, Help subtable: tabelle aiuto per il gestore aiuto PM. Permette al PM di collegare le voci dei dialoghi alle pagine di aiuto; purtroppo, terribilmente difficile da usare.

  • DBCS uniq/font driver directory, DBCS uniq/font driver: programmi di controllo per tipi di carattere double-byte character set (DBCS). Non mi Š chiaro a che serva.

  • String ID table: tabella risorse dei nomi. Non mi Š chiaro a che serva.

  • Application specific: dati in un formato specifico di applicazione.
