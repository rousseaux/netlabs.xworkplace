I quattro criteri di ordinamento seguenti sono sempre disponibili in tutte le &link_folders;:
  • Ordina per nome: si ordina una cartella alfabeticamente usando i nomi degli oggetti.

  • Ordina per tipo: si ordina una cartella usando i tipi degli oggetti, per esempio l'attributo esteso .TYPE, che pu• essere settato nel blocco appunti mpostazioni degli oggetti. Si noti che quanto detto nell'esempio si applica agli oggetti file di dati; tutti gli altri oggetti (e oggetti file di dati senza tipo definito) saranno ordinati per primi.

  • Ordina per classe: si ordina una cartella usando le classi degli oggetti, visibili nella terza colonna della modalit… visualizzazione dettagli delle cartelle.

  • Ordina per estensione nome file: si ordina una cartella usando i caratteri dopo l'ultimo punto nel nome di un oggetto, che normalmente Š l'estensione del nome del file (come .TXT, .EXE ecc.). Gli oggetti senza estensione sono messi per primi.
Sono poi disponibili anche i seguenti criteri, purch‚ la cartella non abbia modificato la propria classe di ordinamento in qualcosa diverso da WPFileSystem (il che di solito accade solo con cartelle speciali, come il cestino e la cartella "Tipi di carattere"):
  • Ordina per nome effettivo: si ordina una cartella alfabeticamente secondo il nome fisico nel file-system. PoichŠ solo gli oggetti file hanno nomi reali, gli oggetti astratti non vengono ordinati correttamente.

  • Ordina per dimensione: si ordina una cartella usando le dimensioni dei file. PoichŠ gli oggetti astratti non hanno dimensione (occupano infatti solo spazio nel file OS2.INI) si considera che la loro dimensione sia zero byte.

  • Ordina per data ultima scrittura / data ultimo accesso / data creazione: si ordina una cartella secondo questa informazione del file-system, che non Š disponibile per gli oggetti astratti. Notare che il file system FAT non conserva tutte queste informazioni, perci• questi tre criteri in questo caso non fanno differenza.
Se l'impostazione "Cartelle davanti" Š abilitata, le cartelle saranno disposte comunque prima degli altri oggetti.

All'interno di un gruppo con cartelle e oggetti diversi dalle cartelle, l'ordinamento segue il criterio relativo. Per esempio, se Š selezionato "Ordina per data ultima scrittura" e si Š abilitato anche "Cartelle davanti", verranno messe per prime tutte le sottocartelle, ordinate per data di ultima scrittura, e poi tutti gli altri oggetti, anch'essi ordinati per data di ultima scrittura.
