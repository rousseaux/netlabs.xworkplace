Starting with V1.0.0, &xwp; requires at least OS/2 Warp 4. It will not run on previous versions, for it calls OS/2 API functions not present on OS/2 Warp 3 and before.

&xwp; is currently being developed and tested on eComStation 1.0 only, but should work on Warp 4 as well. I do not think it relies on functionality that is present only in newer fixpaks (especially since the IBM source code merge with FP13), but then I didn't test this.
