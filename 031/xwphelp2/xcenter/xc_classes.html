In de "Classes" pagina in het instellingen notitieblok van het &xcenter; ziet u de widget classes die op dit moment beschikbaar zijn.

Let op dat widget classes geen echte WPS classes zijn. Het &xcenter; gebruikt alleen de class namen om de widgets intern te organiseren. Beschouw een widget class als een model waarvan widgets gemaakt kunnen worden. Een widget class is ofwel ingebouwd in het &xcenter; zelf ofwel geladen vanuit een plug-in DLL uit de ecs\system\ewps\plugins\ecenter WPS\XWorkplace\plugins\xcenter directory. Zo'n plug-in DLL kan een of meer widget classes bevatten.

In tegenstelling tot de "Widgets" pagina, welke de huidige widgets laat zien van een enkel &xcenter;, laat de "Classes" pagina dezelfde widget classes zien voor elke &xcenter;. De classes die hier worden getoond, zijn ook te vinden in het "Voeg widget toe" submenu van het &popmenu; van een open &xcenter;.

De volgende velden worden getoond:

  • Het "DLL" veld toont de naam van de Dynamic Link Library (DLL) welke de widget implementeerde. Dit is ofwel ‚‚n van de DLL's uit de WPS\XWorkplace\plugins\xcenter ecs\system\ewps\plugins\ecenter directory ofwel zal het de tekst "Ingebouwde" laten zien als het widget opgenomen is in &xwp; zelf.

  • De "Widget class" laat de interne naam van de class zien die gebruikt wordt door het &xcenter; om de class, waar een widget bij hoort, te identificeren. Deze naam is kritisch voor de werking van een widget en zal nooit veranderen, tenzij de DLL niet langer beschikbaar is.

  • Het "Class naam" veld heeft dezelfde inhoud als het "Voeg widget aan lade toe" menu en is een beschrijvende naam van de class. Dit is alleen voor informatieve doeleinden en niet kritisch voor het werken van de widget.

  • Het "Versie" veld toont de minimale &xcenter; versie die de widget nodig heeft om te functioneren. Het &xcenter; controleert dit versienummer om er zeker van te zijn dat het de plug-in DLL kan laden. Dit veld is niet van betekenis voor ingebouwde widgets en is daarom, voor die widgets, leeg.
