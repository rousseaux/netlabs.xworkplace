De volgende vier sorteer criteria zijn altijd beschikbaar in alle &link_folders;:
  • Sorteren op "Naam": dit sorteert een map alfabetisch overeenkomstig de namen van de objecten.

  • Sorteren op "Type": dit sorteert een map overeenkomstig de object typen, d.w.z. het uitgebreide kenmerk TYPE dat kan worden ingesteld in het instellingen notitieblok van een object. Merk op dat dit alleen geldt voor objecten van data bestanden; alle andere objecten (en objecten van data bestanden van een ongedefinieerd type) zullen aflopend worden gesorteerd.

  • Sorteren op "Object class": dit sorteert de map overeenkomstig de object classes, d.w.z. dat wat zichtbaar is in de derde kolom van een "Details" overzicht van het pictogram van een map.

  • Sorteren op "Bestandsnaam uitbreiding": dit sorteert een map overeenkomstig de karakters na de laatste punt in de naam van een object dat normaliter een bestandsnaam uitbreiding is (zoals .TXT, .EXE, enz.). Objecten zonder een uitbreiding komen eerst.
Tenzij een map zijn sorteer class heeft gewijzigd naar iets anders dan het WSFileSystem, dat normaliter alleen maar gebeurt met speciale mappen zoals de prullenbak en de "Lettertypen" map, zijn de volgende sorteer criteria eveneens beschikbaar:
  • Sorteren op "Naam": (dat is de "Echte naam ") dit sorteert een map alfabetisch overeenkomstig de bestandssysteem namen. Omdat alleen bestandssysteem objecten echte namen hebben, zullen onbenoemde objecten niet juist worden gesorteerd.

  • Sorteren op "Grootte": dit sorteert een map overeenkomstig de grootte van de bestanden. Omdat onbenoemde objecten geen bestandsgrootte hebben, maar alleen wat ruimte innemen in OS2.INI, wordt aangenomen dat zij een grootte hebben van nul bytes.

  • Sorteren op "Gemaakt op:" / "Laatst gewijzigd op:" / "Laatst geopend op:": dit sorteert de map overeenkomstig de bestandssysteem informatie, die niet beschikbaar is voor onbenoemde objecten. Merk op dat het FAT bestandssysteem al deze informatie niet ondersteunt, dus zullen deze drie criteria daar voor geen verschil zorgen.
Als de "Mappen eerst" instelling ook is ingeschakeld, worden alle mappen aan de bovenzijde in de map geplaatst.

In de groep 'mappen' en 'objecten die geen mappen' zijn, wordt het sorteren overeenkomstig de huidige sorteer criteria gedaan. B.v., als "Sorteren" op "Laatst gewijzigd op: (datum)" wordt geselecteerd en "Mappen eerst" ook is geselecteerd, worden alle submappen aan de bovenzijde in de map geplaatst, alle submappen worden dan gesorteerd op laatst in geschreven datum, en alle objecten die geen map zijn worden ook op dat gegegeven gesorteerd.
