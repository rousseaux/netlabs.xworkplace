De "Hulpbronnen" pagina komt in alle instellingen notitieblokken voor van alle uitvoerbare bestanden om extra informatie te laten zien van de uitvoerbare module.

Hulpbronnen zijn alleen-lezen gegevens objecten die dynamisch kunnen worden benaderd tijdens uitvoering.

Hulpbron objecten worden in een uitvoerbaar bestand geplaatst door de 'Resource Compiler', die met &os2; meegeleverd wordt. Dit kan of het hoofdonderdeel van de applicatie zijn (.EXE bestand) of een dynamic link library (DLL).

Een hulpbron kan op ieder moment worden geladen door het gebruik van een "type" en "ID" aanduiding. Typisch in tegenstelling tot de feitelijke applicatie code is, dat hulpbronnen alleen worden geladen als zij ook werkelijk nodig zijn. Dat maakt het starten van een applicatie sneller en vermindert ook het gebruik van geheugen.

Hulpbronnen worden veelvuldig door applicaties gebruikt om hun taal ondersteuning (NLS) te scheiden van de feitelijke code. Op deze manier, kunnen applicaties worden vertaald zonder dat de bron code opnieuw gecompileerd behoeft te worden !&xwp; doet zijn taal ondersteuning b.v. op deze manier.)

De volgende typen hulpbronnen bestaan:

  • Muis aanwijzers: een pictogram of aanwijzer bestand (.ICO/.PTR formaat).

  • Bitmap: een OS/2 bitmap (.BMP)

  • Menu model: een PM menu definitie; of een menu balk of een voorgrond (context) menu.

  • Dialoog model: een PM dialoog definitie.

  • Tekst tabel: een tekst tabel.

  • Lettertype directory, Lettertype: PM bitmap lettertypen.

  • Versneller tabel: een PM versneller (sneltoets, toetsenbord verkorte aanslag) tabel.

  • Binaire gegevens: binaire gegevens in een ongedefinieerd formaat.

  • Fout boodschappen tabel: een tabel met fout boodschappen.

  • Dialoog bestandsnaam invoegen: bestandsnaam van de dialoog bestand invoegen.

  • Hulp tabel, Hulp subtabel: hulp tabellen voor de PM hulp manager. Dit maakt het mogelijk dat PM dialoog onderdelen en hulp pagina's samen brengt, maar is erg hinderlijk in gebruik.

  • Applicatie specifiek: gegevens in een applicatie-specifiek formaat. Van de volgende aanwezige hulpbronnen is, door een gebrek aan oorspronkelijke informatie, niet precies bekend wat zij betekenen

  • Virtuele toets tabel: sleutel naar de VKEY tabellen.

  • Key table: sleutel naar UGL tabellen.

  • Karakter tabel: Hi‰roglief naar karakter tabellen. Dit wordt waarschijnlijk bij lettertypen gebruikt.

  • Scherm informatie: scherm weergave informatie.

  • Short-form function key area: function key area, short form.

  • Long-form function key area: function key area, long form.

  • DBCS uniek/lettertype stuurprogramma directory, DBCS unieke/lettertype stuurprogramma: double-byte karakter set (DBCS) lettertype stuurprogramma's.

  • Reeks ID tabel: hulpbronnen naam tabel.
