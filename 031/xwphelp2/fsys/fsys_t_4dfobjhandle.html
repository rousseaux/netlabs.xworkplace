De "Sla gegevensbestand object ingang, in omgeving, over" instelling bepaalt hoe &xwp; de WP_OBJHANDLE variabele zal instellen in de omgeving van een programma dat werd gestart via een gegevensbestandskoppeling. Uitschakelen van deze instelling maakt het u mogelijk om het aantal bestandssysteem object ingangen te verminderen die op uw systeem worden gemaakt.
  • Als u een programma start via een gegevensbestandskoppeling, en de instelling is aangevinkt, zullen de object ingangen van de gegevensbestanden in de omgeving ook worden overgeslagen. De WP_OBJHANDLE zal dan als volgt worden ingesteld:

    WP_OBJHANDLE=programma,gegevensbestand

    waarin programma de object ingang is van het programma object of programmabestand dat werd gestart en gegevensbestand de object ingang is van het gegevensbestand waarvan de koppeling het starten van het programma veroorzaakte.

    Dit is het standaard gedrag van de Workplace Shell.

  • Als de instelling niet is aangevinkt, zal de WP_OBJHANDLE als volgt worden ingesteld:

    WP_OBJHANDLE=programma

    M.a.w., de gegevens bestandsingang zal niet worden overgeslagen

Deze instelling werd toegevoegd omdat iedere keer als een gegevensbestand wordt gestart, b.v. wanneer u dubbelklikt op een gegevensbestand, er een object ingang voor het bestand moet worden gemaakt. Die object ingang blijft voor altijd in het OS2SYS.INI bestand staan, zelfs als het bestand nooit meer gebruikt wordt. Uiteindelijk zal dit een groot aantal ingangen cre‰ren. U kunt dus deze instelling nu uitzetten en u zult zien dat de ingangen sectie in OS2SYS.INI niet zo snel meer zal groeien.

Echter, bepaalde applicaties zijn afhankelijk van het standaard Workplace Shell gedrag. Als u deze instelling uitschakelt en een applicatie werkt niet meer, moet u de instelling weer inschakelen.

Deze instelling werkt onmiddellijk.
