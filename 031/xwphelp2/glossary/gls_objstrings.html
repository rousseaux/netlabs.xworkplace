Om de instellingen van een object van "buiten" de Workplace Shell te kunnen wijzigen, ondersteunt de WPS het concept van object instellingen reeksen.

Een WPS instellingen reeks is een opeenvolging van keyword=value (sleutelwoord=waarde) paren, gescheiden door een punt-komma ';'. De steekwoorden die door een object worden begrepen zijn afhankelijk van zijn class.

B.v., om van een map de gesloten en open pictogrammen te wijzigen, kunt u de volgende tekenreeks gebruiken:

ICONFILE=C:\closed.ico;ICONNFILE=1,C:\open.ico;

Omdat de WPS classes gegroepeerd zijn in een hi‰rarchische boomstructuur en van elkaar overerven, zal een object niet alleen de instellingen reeks van zijn eigen class begrijpen maar ook die van al zijn ouder classes. B.v., een map zal niet alleen WPFolder steekwoorden begrijpen, maar ook de algemene WPObject sleutelwoorden (die door alle objecten worden begrepen).

Er zijn zeer veel verschillende object steekwoorden voor de verschillende WPS classes. Kijk voor meer informatie in de onderstaande onderwerpen:

  • WPObject (alle objecten)
  • WPFolder (alle mappen)
  • WPDesktop (de Werkplek)
  • XWPTrashCan (de prullenbak)
  • WPProgram (alle programma objecten)
  • XCenter (het &xcenter;)
  • XWPString (objecten instellingen reeks)
Er zijn verschillende manieren om instellingen reeksen toe te passen op een object.
  • &xwp; staat u toe om een object instellingen reeks te gebruiken.

  • Uit &xwp; REXX kunt u de functie SysSetObjectData gebruiken of, als u een nieuw object aan het maken bent, de functie SysCreateObject.

  • Uit de C code kunt u de PM WinSetObjectData en WinCreateObject API's gebruiken, zie: *.

  • Vanuit een WPS class kunt u de WPObject::wpSetup methode gebruiken die veel subclasses te niet doet en de werkelijke instellingen reeks verwerking implementeert binnen de WPS, zie: *.
Opmerking: een populaire misvatting is dat, als een instellingen reeks wordt aangeroepen door een object, de reeks zelf nergens wordt opgeslagen. In plaats daarvan wordt de reeks verwerkt door de WPS class van het object, vertaald in de binaire gegevens van het object, en dan verwijderd.

Met als resultaat dat &xwp;, om in staat te zijn de instellingen reeksen weer op te zoeken om ze te tonen in de "Pictogram" pagina van het instellingen notitieblok van een object, de binaire gegevens van een object moet evalueren en de juiste beslissing moet nemen wat de instellingen reeks zou kunnen zijn die de binaire instelling zou moeten maken. Opmerking: van de object gegevens is totaal niets beschreven en ondanks dat er een weg beschreven is om er achter te komen wat de binaire object gegevens betekenen, is &xwp; niet in staat om van daaruit een instellingen reeks te maken
