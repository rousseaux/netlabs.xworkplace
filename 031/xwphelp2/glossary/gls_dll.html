Een dynamic link library is een speciaal soort uitvoerbaar bestand dat code bevat die door verschillende applicaties kan worden gedeeld. Dynamic link libraries hebben meestal een .DLL bestandsnaam uitbreiding.

DLL's kunnen of met het besturingssysteem of met een applicatie meekomen. De systeem code van &os2; zelf is ook in grote mate verzameld in DLL's en kan worden gevonden in de OS2\DLL directory op de opstart schijf.

"Dynamisch linken" betekent dat de code uit de module alleen wordt geladen als een bepaalde applicatie er om vraagt. Als de code reeds geladen was door een andere applicatie wordt het hergebruikt. Dit heeft verschillende voordelen:

  • Diverse applicaties kunnen werkelijk de code delen van een DLL, wat zowel schijf ruimte bespaart als hoofdgeheugen van uw computer.

  • Applicaties kunnen als modules gemaakt worden. B.v., het taal afhankelijke deel (NLS) voor een DLL kan in een aparte DLL zitten. Dit maakt wijzigingen van de taal in een applicatie mogelijk door de wisseling van een enkel bestand. DLL's kunnen ook de software ondersteuning vereenvoudigen omdat dan het opnieuw zenden van een complete applicatie niet zo vaak nodig is om fouten te herstellen.

  • DLL's kunnen recente subsystemen bevatten die door verschillende applicaties worden gebruikt, omdat een DLL gegevens kan delen tussen processen. Dit maakt b.v. eenvoudigere gevens uitwisseling mogelijk en is ook ‚‚n van de redenen waarom grote delen van &os2; zelf zijn opgeslagen in DLL's.

  • DLL's kunnen worden gebruikt om een applicatie uit te breiden.
