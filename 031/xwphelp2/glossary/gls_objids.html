De Workplace Shell staat toe dat ieder object naar keuze een object ID heeft, dat als enige het object in het systeem identificeert. Alle objecten kunnen zulke ID's hebben, onafhankelijk van welke class.

Per definitie moeten de object ID's van de Werkplek een karakter reeks tussen vierkante haken zijn. B.v. de standaard &link_syssetupfdr; heeft de <WP_CONFIG> object ID. Door gebruik te maken van een object ID, kunnen applicaties refereren aan objecten zonder te weten in welke map zij op dat moment staan.

Veel object ID's zijn toegewezen door het &os2; installatie programma toen het systeem voor het eerst is ge‹nstalleerd en gewoonlijk nadien nooit veranderd. (Deze ID's moeten in de eerste plaats niet worden veranderd omdat diverse applicaties of de WPS zelf kunnen steunen op het ID om het object te identificeren.)

Er zijn echter gewoonlijk veel meer object ID's op uw systeem dan die toegewezen zijn bij de eerste &os2; installatie. Veel installatie programma's zullen ook ID's toewijzen aan hun objecten om in staat te zijn ze later weer te kunnen verwijderen.

&xwp; staat u toe om het object ID te bekijken en te wijzigen in de "Pictogram" pagina van elk instellingen notitieblok. Echter, het wijzigen van object ID's moet met de grootst mogelijke zorg gebeuren.
