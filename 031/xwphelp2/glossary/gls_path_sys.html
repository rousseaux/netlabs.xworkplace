&os2; heeft verschillende systeempaden die aangeven waar het systeem naar bestanden moet zoeken als met een bestand geen pad is meegegeven.

Bijvoorbeeld, als een uitvoerbaar bestand in de lijst met directory's staat aangeduid met zijn systeempad (in PATH in de config.sys in de root directory van &OS2;), wordt het uitvoerbare bestand gevonden zonder excact te weten waar het in het systeem is opgeslagen. Daarom kunt u eenvoudig e intikken achter een &os2; opdrachtaanwijzing om de &eexe; op te starten: E.EXE bevindt zich in een directory op uw opstartstation, die in het PATH staat.

Uitzondering is het LIBPATH, dit zijn &xwp; omgevingsvariabelen die eerst worden ingesteld door het &cfgsys; bestand, bij het opstarten van het systeem en die dus later voor iedere sessie kunnen worden veranderd.

U kunt de huidige systeempad instellingen bewerken op de "Systeempaden" pagina in het "&os2; Kernel" object.

De volgende systeempaden zijn gedefinieerd:

  • LIBPATH: het "bibliotheek" systeempad, dat elke keer wordt doorzocht als &os2; tracht een &xwp; Dynamic Link Library (DLL) te laden. Deze bevat zowel de &os2; systeemdirectory's als een punt (".") ingang, wat staat voor de huidige directory.

    Zie de documentatie in de Handleiding bij OS/2 opdrachten voor details.

    Merk op dat dit het enige systeempad is dat niet is gedefinieerd om gebruik te maken van een omgevingsvariabele en daarom niet kan worden veranderd vanaf de opdrachtregel. Gebruik de BEGINLIBPATH en ENDLIBPATH variabelen voor tijdelijke wijzigingen.

  • SET PATH: het zoekpad voor uitvoerbare (.EXE, .COM, .BAT en .CMD) bestanden.

    Zie de documentatie in de Handleiding bij OS/2 opdrachten voor details.

  • SET DPATH: het zoekpad voor gegevensbestanden. Het gebruik van dit zoekpad hangt van de applicaties af. Sommige applicaties gebruiken het, anderen niet.

    Zie de documentatie in de Handleiding bij OS/2 opdrachten voor details.

  • SET BOOKSHELF: dit wordt door de VIEW.EXE gebruikt om digitale .INF boeken te vinden. Hoewel elk .INF boek kan worden bekeken, zelfs als het niet in dit zoek pad zit, zal de "Overal zoeken" functie van VIEW.EXE alleen onderdelen vinden in boeken die in dit pad aanwezig zijn.

  • SET HELP: dit wordt door de PM helpfaciliteit gebruikt om help panelen (.HLP bestanden) te tonen. Sommige applicaties geven niet aan PM het volledige pad op van de helpbestanden om help panelen te kunnen bekijken. PM verlangt dat de helpbestanden in dit pad aanwezig zijn. &xwp; heeft zo'n ingang ook niet nodig.
