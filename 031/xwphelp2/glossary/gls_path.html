Ieder bestandssysteem object (bestand of map) heeft een koppeling met een pad, dat het systeem aangeeft waar het object is opgeslagen zodat het vanuit applicaties kan worden gevonden.

Het pad bestaat uit alle directories die moeten worden geopend om bij het betreffende bestand uit te komen. (Directories zijn de fysieke representaties van de &link_folders; op de schijf.

Een volledig geldig pad bestaat uit de schijfnaam, gevolgd door een dubbele punt (:) een aanvangs- 'backslash' (\) om de hoofd directory aan te geven, plus een lijst van directories die moeten worden geopend om bij het object te komen, allemaal gescheiden door weer een 'backslash' (\).

Als het pad naar een bestand wijst, b.v een gegevens bestand of programma bestand, wordt hij gevolgd door de bestandsnaam. B.v., de &os2; &eexe; bevindt zich in de OS2 directory van de opstart schijf. Het volledige pad is dan: 

    X:\OS2\E.EXE
waar X: staat voor de schijfletter van uw opstartschijf.

&os2; heeft verschillende systeem paden die aangeven waar het systeem moet kijken voor bestanden als een bestand niet voorzien is van een pad aanduiding. Zie "Systeem paden" voor een nadere verklaring.
