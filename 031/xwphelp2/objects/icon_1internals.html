Het "Details:" veld laat u veel interne object gegevens zien waar u wel of niet in ge‹nteresseerd bent. Veel hiervan is voornamelijk interessant voor programmeurs.

Deze informatie is gegroepeerd in een boomstructuur, die op een vergelijkbare manier als een mapstructuur weergave te gebruiken is. Echter, de meeste informatie in deze map is alleen-lezen, oftewel kan niet aangepast worden (uitgezonderd het "Object ID" gegeven).

De "Object ingang" is een vijfcijferig hexadecimaal getal dat het object uniek identificeert op uw systeem. Abstracte objecten krijgen altijd object ingangen, terwijl bestandssysteem objecten die niet altijd hoeven te hebben. Denk er om dat object ingangen niet gebruikt kunnen worden om objecten te identificeren op andere dan uw eigen systeem: dat kan niet. Ga er bijvoorbeeld eens vanuit dat de "Informatie" map op het systeem van uw buurman dezelfde object ingang heeft als die op uw systeem. Object ingangen verwijzen alleen naar objecten op uw computer.

Zie: Object ingangen voor een introductie.

Het "Object ID" kan echter ook op andere computers bestaan. Object ID's hebben altijd het volgende formaat <OBJECTID> en worden gebruikt om naar objecten te verwijzen zonder te weten waar ze in werkelijkheid staan. Zie: Object IDs voor een introductie.

Het "Object ID" veld kan aangepast worden door er op te klikken met de "Alt" toets ingedrukt. Wees echter voorzichtig met het wijzigen van object ID's. Sommige programma's kunnen afhankelijk zijn van bepaalde ID's.

De "Objectstijl" boomstructuur laat u WPS-interne objectvlaggen zien. De objectstijl vlaggen bepalen alleen het WPS gedrag, bijvoorbeeld, als de "Niet kopi‰ren" vlag is geselecteerd, dan kunt u, vanaf de WPS, geen sleep-kopieer actie op het bestand doen. Denk er om dat deze vlaggen normaliter los staan van de &xwp; Bestandskenmerken bits, oftewel een bestand kan de "niet verwijderen" vlag hebben staan, maar kan vervolgens toch via de opdrachtregel verwijderd worden.
