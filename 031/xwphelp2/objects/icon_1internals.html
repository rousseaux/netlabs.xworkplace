De "Object Details" container laat u veel interne object gegevens zien waar u wel of niet in ge‹nteresseerd bent. Veel hiervan is voornamelijk interessant voor programmeurs.

Deze informatie is gegroepeerd door middel van een container boomstructuur, die op een vergelijkbare manier als een mapstructuur overzicht te gebruiken is. Echter, de meeste informatie in deze map is alleen-lezen, oftewel kan niet aangepast worden (uitgezonderd het "Object ID" gegeven).

De "Object handle" is een vijfcijferig hexadecimaal getal dat het object uniek identificeert op uw systeem. Abstracte objecten krijgen altijd object handles, terwijl bestandssysteem objecten die niet altijd hoeven te hebben. Denk er om dat object handles niet gebruikt kunnen worden om objecten te identificeren op andere dan uw eigen systeem: dat kan niet. Ga er bijvoorbeeld eens vanuit dat de "Informatie" map op het systeem van uw buurman dezelfde object handle heeft als die op uw systeem. Object handles verwijzen alleen naar objecten op uw computer.

Zie Object Handles voor een introductie.

Het "Object ID" kan echter ook op andere computers bestaan. Object ID's hebben altijd het volgende formaat <OBJECTID> en worden gebruikt om naar objecten te verwijzen zonder te weten waar ze in werkelijkheid zijn. Zie object IDs voor een introductie.

Het "Object ID" veld kan aangepast worden door er op te klikken met de "Alt" toets ingedrukt. Wees echter voorzichtig met het wijzigen van object ID's. Sommige programma's kunnen afhankelijk zijn van bepaalde ID's.

De "Object stijl" boomstructuur laat u WPS-interne object vlaggen zien. De object stijl vlaggen bepalen alleen het WPS gedrag, bijvoorbeeld, als de "Niet kopi‰ren" vlag geselecteerd is, dan kunt u geen sleep-kopieer actie voor het bestand doen vanaf de WPS. Denk er om dat deze vlaggen normaliter los staan van de bestandskenmerken, oftewel een bestand kan de "niet verwijderen" vlag hebben staan, maar kan vervolgens toch via de opdrachtregel verwijderd worden.

De "Object gebruik" boomstructuur laat u extra interne object gegevens zien. Deze gegevens zijn voornamelijk handig voor programmeurs en geven de informatie weer die terugkomt van de wpFindUseItem methode.*

&xwp; voegt bij mappen ook de "Map gegevens" boomstructuur toe. Deze bevat vlaggen die gegevens vertegenwoordigen die teruggegeven worden bij de wpQueryFldrFlags* methode, bijvoorbeeld of de map opgebouwd is of niet.
