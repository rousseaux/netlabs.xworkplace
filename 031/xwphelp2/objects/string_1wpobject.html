WPObject is de WPS basis class voor alle objecten. Hij wordt niet direct gebruikt, maar alle andere WPS classes zijn er van afgeleid (zie: WPS classes voor details). Als gevolg daarvan, ondersteunen alle Werkplek objecten (inclusief bestanden, mappen en programma objecten) de volgende instellingen reeksen:
CCVIEW=DEFAULT
Gebruikt de standaardwaarde van de gezamenlijke weergave instelling op het systeem, als de gebruiker "openen" kiest. Dit is de standaardwaarde.

CCVIEW=YES
Maakt een nieuwe weergave van het object, telkens wanneer de gebruiker "openen" kiest.

CCVIEW=NO
Heropent een weergave van het object, als de gebruiker openen kiest.

HELPLIBRARY=bestandsnaam
Stelt de help bibliotheek in.

HELPPANEL=id
Stelt het object zijn standaard help paneel in binnen de help bibliotheek. U kunt beter niet rommelen met deze instellingen, want u dient de WPS interne help bronnen ID's te kennen.

HIDEBUTTON={YES|NO}
Bepaalt of een weergave van dit object een verberg knop heeft in plaats van een minimaliseer knop. De standaard is de huidige systeem knoppen weergave voor de venster instellingen. Dit kan hier niet opgegeven worden.

ICONFILE=bestandsnaam
Bepaalt het object zijn pictogram bestandsnaam. Moet een ".ICO" bestand zijn.

Denk er om dat dit voor mappen het "gesloten" pictogram betreft. Het bewegende ("open") pictogram kan ingesteld worden met ICONNFILE=index,bestandsnaam (zie: WPFolder reeksen voor details).

ICONPOS=x,y
Bepaalt van het object zijn eerste pictogram positie in de bron map. De x en y waarden vertegenwoordigen de positie in de map in procentuele co”rdinaten.

ICONRESOURCE=id,dllfile
Stelt het pictogram van het object in op gegevens die zich in een bron bestand bevinden. De bron id van het pictogram is het id in de DLL dllfile.

Denk er om dat dit voor mappen het "gesloten" pictogram betreft. Het bewegende ("open") pictogram kan ingesteld worden met ICONNRESOURCE=index,id,dllfile (zie WPFolder reeksen voor details).

LOCKEDINPLACE={YES|NO} (alleen &warp4;)
Geeft aan of een object vast of niet vast staat op een positie.

MENUITEMSELECTEDmenu-item-id (alleen &warp4;)
Simuleert de selectie van het &popmenu; van de opgegeven menu optie van het object. menu-item-id moet een numerieke waarde zijn.
Dit is niet echt een instelling, maar meer een opdracht die uitgevoerd kan worden op een map.

MENUS={SHORT|LONG} (alleen &warp4;)
Bepaalt of dit object wel of niet een kort &popmenu; heeft (zonder menu regels die sleep en loslaat acties dupliceren) of alleen het volledige.

MINWIN={HIDE|VIEWER|DESKTOP}
Bepaalt of het geminimaliseerde object wel of niet verborgen kan worden, in de "Vensters als pictogrammen" weergave, of als pictogram op de Werkplek terecht komt.

De standaardwaarde is de huidige minimaliseer knop instelling. Die kan hier niet opgegeven worden.

NOCOPY={YES|NO}
Het object kan niet gekopieerd worden.

NODELETE={YES|NO}
Het object kan niet verwijderd worden. Wees hier voorzichtig mee.

NODRAG={YES|NO}
Het object kan niet versleept worden met de muis.

NODROP={YES|NO}
Het object zal geen objecten accepteren die er op los gelaten worden.

NOLINK={YES|NO}
Er kunnen geen reflecties van dit object gemaakt worden.

NOMOVE={YES|NO}
Het object kan niet verplaatst worden.

NOPRINT={YES|NO}
Het object kan niet afgedrukt worden.

NORENAME={YES|NO}
Het object kan niet een andere naam gegeven worden.

NOSETTINGS={YES|NO}
Het object zijn instellingen notitieblok kan niet geopend worden.

NOSHADOW={YES|NO}
Zelfde als NOLINK.

NOTVISIBLE={YES|NO}
Het object is niet zichtbaar. Wees voorzichtig met deze stijl, want u zou het object wel eens niet meer terug kunnen krijgen.

OBJECTID=<name>
Stelt een blijvend ID in voor het object. Een Object ID is een unieke reeks voorafgegaan door een < en afgesloten met een >.
Zie: WPS Object ID's voor meer over dit onderwerp.

De standaard is om geen ID te hebben.

OPEN={SETTINGS|DEFAULT|ICON|TREE|DETAILS}
Open nu een weergave.
OPEN=DEFAULT zal de standaard weergave van het object openen (alsof u erop dubbelgeklikt had). OPEN=SETTINGS zal het instellingen notitieblok van het object openen. Alle andere waarden zijn alleen geldig voor mappen.
Dit is niet echt een "instelling", maar meer een opdracht die uitgevoerd kan worden op een map. Die zal dan direct openen.

TEMPLATE={YES|NO}
Bepaalt of dit object een model is of niet. De standaardwaarde is (natuurlijk) NO.

TITLE=naam
Stelt de object naam in op iets nieuws. Dit zal ook de "echte" bestandsnaam aanpassen (als het object een bestand vertegenwoordigt), net alsof u de naam gewijzigd heeft op de Werkplek.
