XWPString verzorgt instellingen tekst objecten. Ook al is de XWPString class ontworpen om instellingen teksten op te slaan, kunt u ook instellingen teksten op een XWPString object zelf toepassen, evenals elk ander object.

In aanvulling op de instellingen teksten die ondersteund worden door de WPObject class, ondersteund de XWPString class de volgende teksten:

SETUPSTRING=tekst

De instellingen tekst die in het tekst object opgeslagen moet worden. tekst mag niet opgebouwd zijn uit karakters die niet toegestaan zijn in Desktop instellingen tekst waarden. De standaard is een lege tekst.

DEFAULTOBJECT={objectid|bestandsnaam|*handle|NONE}

Zet of verwijderd het standaard object dat opgeslagen is in het tekst object. De waarde moet een van de volgende zijn:
  • een object ID, omsloten door kleiner dan/groter dan teken (b.v. DEFAULTOBJECT=<WP_DESKTOP>;);
  • een volledig opgegeven bestandsnaam (b.v. DEFAULTOBJECT=C:\mijnbestanden\iets.txt;);
  • een decimale object handle, voorafgegaan door een sterretje (b.v. DEFAULTOBJECT=*1432;);
  • de NONE code om het huidige standaard object te verwijderen, als die er is.
De standaard voor een tekst object is om geen standaard object te hebben(NONE).

CONFIRMINVOCATION={YES|NO}

Komt overeen met de "Bevestig implementatie" instelling op de "Tekst" notitieblok instellingen pagina. De standaard is YES.

NOSTRINGPAGE={YES|NO}

Als YES is, zal het tekst object instellingen notitieblok de "Tekst" pagina niet tonen. Dit kan handig zijn als u tekst objecten in uw eigen programma maakt en u wilt voorkomen dat de gebruiker per ongeluk een zorgvuldig ontworpen instellingen tekst veranderd. De standaard is NO.
