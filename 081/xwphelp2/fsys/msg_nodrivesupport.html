This message appears when you have attempted to delete an object, but the drive on which the object resides is not supported by the &xwp; trash can.

This message means that the object is on a drive with a file system that the trash can does not enable support for by default. This is most frequently the case for remote network drives and more exotic installable file system drivers where the trash can cannot be sure whether it can write to the file system without provoking errors.

Do one of the following:

  • Press "Yes" in the confirmation dialog to have the object permanently deleted without moving it into the trash can first. Note that this operation can probably not be undone; if you delete the wrong object, it will be gone anyway.

  • If you are sure that the trash can can support the file system on which the object resides, you can enable support for the drive in the properties notebook of the trash can.
